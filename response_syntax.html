ORIZIN Agent は、オープンソースの AI アシスタントです。


認識する言葉のカスタマイズ


1. 基本

基本的には、response.txtを編集することで挙動をカスタマイズすることができます。以下に、具体例を示します。

オリジン:オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。
おりじん:オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。
orizin:オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。
origin:オリジンは、私の名前の元となった、原点という意味の英単語です。また、私の名前の綴りはORIZINで、Gの部分がZになっています。
  • :の前が反応する言葉、あとが応答です。例えば1行目であれば、入力内容にオリジンという言葉が含まれていたらオリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。と応答するという意味です。
  • :は半角を使用してください。
  • 反応する言葉と応答1セットごとに改行を入れてください。また、空行は入れないでください。
  • プログラム中でユーザーからの入力に含まれる空白、・、_等は取り除かれ、アルファベットは小文字に変換されます。そのため、反応する言葉は、空白、・、_を取り除いた（アルファベットは）小文字で書く必要があります。
Tips!

/を使うことでresponse.txtを短く、詳しく書くことができます。以下は、上の例と同じ意味です。

オリジン/おりじん/orizin:オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。
origin:オリジンは、私の名前の元となった、原点という意味の英単語です。また、私の名前の綴りはORIZINで、Gの部分がZになっています。

この例は、反応する言葉を複数指定しています。ただし、/は半角である必要があります。また、下の例を見てください。

オリジン/おりじん/orizin:オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。/ORIZINは、私の名前です。原点という意味の英単語、originに由来しています。
origin:オリジンは、私の名前の元となった、原点という意味の英単語です。また、私の名前の綴りはORIZINで、Gの部分がZになっています。/originは、私の名前の元となった、原点という意味の英単語です。また、私の名前の綴りはORIZINで、Gの部分がZになっています。

この例は、例えば1行目であれば、ユーザーからの入力にオリジン、おりじん、orizinのどれかが含まれていた場合、オリジンは、私の名前です。原点という意味の英単語、オリジンに由来しています。と喋り、GUI画面には応答としてORIZINは、私の名前です。原点という意味の英単語、originに由来しています。と表示します。この機能は、英単語を発音することができないOpen JTalkに無理やり英単語を発音させる場合に便利です。今回の例で言えば、ORIZINと喋らせたい場合、通常なら「オー・アール・アイ・ズィー・アイ・エヌ」と発音されてしまうところを、オリジンと発音させ、GUI画面には本来のORIZINを表示させます。

2. 発展

音楽を再生する等、会話以外のことをしたい場合、ORIZIN_Agent.pyを書き換える必要があります。以下は、音楽を再生する例です（ORIZIN_Agent.pyより引用）。

if judge(question, ['catlife', 'キャットライフ']):
    speak('私の好きな曲は、キャットライフです。キャットライフを再生します。', '私の好きな曲は、Cat lifeです。Cat lifeを再生します。')
    playSound(thisDir + '/sounds/musics/wav/catLife.wav')
elif judge(question, ['曲', '音楽', '楽曲', '歌', '唄', 'うた', 'ミュージック']):
    if random.randint(0, 1) == 0:
        speak('私の好きな曲は、キャットライフです。キャットライフを再生します。', '私の好きな曲は、Cat lifeです。Cat lifeを再生します。')
        playSound(thisDir + '/sounds/musics/wav/catLife.wav')
    else:
        speak('私の好きな曲は、せんじんです。せんじんを再生します。', '私の好きな曲は、戦神です。戦神を再生します。')
        playSound(thisDir + '/sounds/musics/wav/senjin.wav')

if judge(question, ['catlife', 'キャットライフ']):は、ユーザーからの入力に'catlife'または'キャットライフ'が含まれているかを判定します。つまり、elif judge(question, ['曲', '音楽', '楽曲', '歌', '唄', 'うた', 'ミュージック']):の部分は、ユーザーからの入力にcatlifeもキャットライフも含まれていない場合に、'曲'、'音楽'、'楽曲'、'歌'、'唄'、'うた'、'ミュージック'の内のどれかが含まれているかを判定します。

speak('私の好きな曲は、キャットライフです。キャットライフを再生します。', '私の好きな曲は、Cat lifeです。Cat lifeを再生します。')は、'私の好きな曲は、キャットライフです。キャットライフを再生します。'と喋り、GUI画面のユーザーの入力とそれに対する応答を表示するスペースの応答に'私の好きな曲は、Cat lifeです。Cat lifeを再生します。'と表示します。また、以下はspeak関数の書き方の例です。
speak('content1', 'content2', request = 'content3', prompt = True, speed = 0)
各引数の意味は、以下の通りです。

引数名 意味 変数の型 省略可/不可 デフォルト値
第1引数 喋らせたい内容 String 不可 なし
第2引数 応答としてGUI画面に表示する内容 String 可('no arg'または省略でデフォルト値が使われる) 第1引数
request ユーザーからの入力として表示する内容 String 可('no arg'または省略でデフォルト値が使われる) ユーザーからの入力
prompt ユーザーからの入力を表示するかどうか Boolean 可(省略するとデフォルト値が使われる) True
speed 話速 0~2のintまたはfloat 可(省略するとデフォルト値が使われる) 1
